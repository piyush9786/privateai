// CAPABILITIES

What's actually wired up

Every item below calls a real backend endpoint — none of this is a mockup.

01 / TOOL-CALLING

Agentic chat

The model decides when to search documents, build a chart, or generate a file mid-conversation, with a live status trail showing each step.

02 / RAG

Private document search

Upload PDFs, spreadsheets, or text. Chunked and embedded into your own ChromaDB instance — never sent anywhere external.

03 / PERSONAS

Custom agent presets

Define named instruction sets and switch between them in chat. Same tools and vault every time, different focus.

04 / ACCOUNTS

Real multi-user isolation

Each account has its own documents, conversations, and memories. Admin role for account management, scoped per user, not shared.

05 / METRICS

Honest analytics

Chat-turn counts and latency recorded at the moment each conversation happens — not estimated, not a placeholder chart.

06 / COMPUTE

Your own GPU

Runs on Ollama with NVIDIA acceleration when available. Llama 3.2, Qwen, and other open models — no API key, no per-token bill.

// TOPOLOGY

How the pieces talk to each other

Five containers on one Docker network. Nothing here calls out to a third party.

browser → nginx → fastapi → {ollama, chromadb}
// HONEST STATUS

What's actually protecting your data

No invented security score. This is genuinely true of the app today.

Password authentication
bcrypt-hashed, never stored in plaintext
Per-user data isolation
Separate documents, chats, and memories per account
httpOnly session cookies
Not readable by page JavaScript
Multi-factor authentication
Not implemented
HTTPS by default
Plain HTTP unless you add a TLS proxy
Air-gapped / no internet
Local inference, but the host still has normal internet access

This is a self-hosted, single-deployment app. If you expose it beyond localhost, add your own TLS termination and consider what additional hardening your threat model needs.

// SETUP

Running in under five minutes

Docker Compose brings up all five services. The first account you create becomes the administrator.

# clone and configure git clone https://github.com/piyush9786/privateai.git cd privateai cp .env.example .env # bring up the stack docker compose up -d docker compose logs -f model-init # once models are ready open http://localhost:3001

Run your own instance

It's a Docker Compose file and an open repository — clone it, build it, point it at your own GPU.
